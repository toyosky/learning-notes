# VLM-Driving-Scene-Understanding 项目分析

## 项目概览

- **项目链接**：https://github.com/Apageoflove/VLM-Driving-Scene-Understanding
- **目标**：用多模态大模型（VLM）理解和分析自动驾驶场景
- **核心思路**：VLM 作为一个具备常识的"老司机"，可以补充传统感知的不足

## 当前辅助驾驶 vs VLM 辅助驾驶

### 当前方案（端到端学习）

```
摄像头/激光雷达 → 深度学习 → 决策规划
```

- 大量数据拟合映射关系
- 缺乏常识推理能力
- 只能处理训练数据中覆盖的场景

### VLM 方案

```
摄像头 + 问题 → VLM → 场景分析和建议
```

- 具备常识推理能力
- 灵活处理各种场景
- 能解释决策原因

## 实现方案

原始 Qwen2.5-VL 无法理解驾驶场景 → 用驾驶场景标注数据做 LoRA 微调 → 微调后能回答"前方有什么车辆""行人意图"等问题

### 数据格式

```json
[
  {
    "id": 0,
    "image": "path/to/image.jpg",
    "conversations": [
      {"role": "user", "value": "描述这个驾驶场景"},
      {"role": "assistant", "value": "前方有一辆白色轿车正在变道..."}
    ]
  }
]
```

## 训练命令

```bash
deepspeed --hostfile=hostfile src/train.py \
  --model_name_or_path Qwen/Qwen2.5-VL-7B-Instruct \
  --dataset json --template qwen2_vl \
  --finetuning_type lora --lora_rank 8 \
  --lora_alpha 16 --lora_target all \
  --output_dir output/vlm_driving
```

### 关键参数

| 参数 | 值 | 含义 |
|------|-----|------|
| lora_rank | 8 | 低秩维度，驾驶场景数据量不大，8 就够 |
| lora_alpha | 16 | alpha = 2 × r，LoRA 更新适度放大 |
| lora_target | all | 所有线性层都微调 |
| template | qwen2_vl | Qwen2.5-VL 的对话模板 |

## 关键技术

1. **SFT 高质量数据** — 让模型学会"看图说话"
2. **视觉编码器** — CLIP / SigLIP 让模型"看得懂"图片
3. **模态对齐** — 投影层让模型理解"图片和文字说的是同一件事"

## 学术方案 vs 工业方案

| | 学术方案（此项目） | 工业方案 |
|--|------------------|---------|
| **架构** | 单一 VLM | 多模型级联 |
| **检测** | VLM 直接回答 | 专用检测头 |
| **优势** | 简单、端到端、可解释 | 精度高、可追溯、低延迟 |
| **劣势** | 精度较低、幻觉风险、延迟高 | 复杂、维护成本高 |

## 评估维度

| 维度 | 评估方式 | 目标 |
|------|---------|------|
| **场景描述准确率** | 人工评分或 GPT-4 评估 | > 90% |
| **目标检测召回率** | 与 Ground Truth 对比 | > 85% |
| **意图判断准确率** | 人类专家标注 | > 80% |
| **推理延迟** | 端到端推理时间 | < 500ms（需优化） |

## 部署考量

1. **模型量化** — INT4 量化可将 7B 模型推理显存降至 ~6GB
2. **视觉编码加速** — 对单帧场景可缓存视觉特征
3. **边缘部署** — 需要 TensorRT / ONNX 等加速框架
4. **Fallback 机制** — VLM 推理失败时回退到传统感知
