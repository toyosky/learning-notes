# 大模型内部结构

## 一句话理解

大模型 = **一堆矩阵** + **一层一层串联**

一个 7B 模型 ≈ 80 层 Transformer，每层都有 Attention 和 FFN 子层。

## 一个 Transformer Block 里有什么

```
输入 → [LayerNorm] → Attention（4个矩阵）→ [残差连接] → [LayerNorm] → FFN（3个矩阵）→ [残差连接] → 输出
```

### Attention 层（理解语义）

```
q_proj: 输入 → 查询向量（你在找什么？）
k_proj: 输入 → 键向量（我能提供什么？）
v_proj: 输入 → 值向量（我的具体内容是？）
o_proj: 注意力结果 → 输出投影
```

核心计算：`Attention(Q,K,V) = softmax(Q × K^T / √d) × V`

- Q × K^T 计算每个 token 对其他 token 的"关注度"
- softmax 归一化为概率分布
- 乘以 V 得到加权后的表示

### FFN 层（记忆知识）

```
gate_proj: 输入 → 门控信号（哪些信息重要？）
up_proj:   输入 → 扩展到高维（提取特征）
down_proj: 压缩回低维 → 输出

FFN输出 = down_proj(silu(gate_proj(x)) * up_proj(x))
```

**知识主要存在 FFN 的权重矩阵里**。FFN 层参数约占模型总参数的 2/3。

## 常见模型架构对比

| 架构 | 代表模型 | 特点 |
|------|---------|------|
| **GPT 式（Decoder-only）** | GPT-4, LLaMA, Qwen, DeepSeek | 因果注意力，只看到上文，适合生成 |
| **BERT 式（Encoder-only）** | BERT, RoBERTa | 双向注意力，适合理解 |
| **T5 式（Encoder-Decoder）** | T5, Flan-T5 | 编码器+解码器，适合翻译/摘要 |

> 当前主流大语言模型几乎全部采用 **Decoder-only** 架构。

## 参数量分布（以 LLaMA-7B 为例）

| 组件 | 参数量（约） | 占比 |
|------|-------------|------|
| Attention（全部层） | ~2.0B | 29% |
| FFN（全部层） | ~4.6B | 66% |
| Embedding + LM Head | ~0.4B | 5% |

## LoRA 微调选择不同层的效果

- **Attention 层**：改变"模型怎么理解语言"——影响风格、格式、对齐
- **FFN 层**：改变"模型记住了什么知识"——影响事实性知识注入

| 目标 | 主要微调对象 | 推荐 r |
|------|-------------|--------|
| 风格/格式调整 | Attention 层的 q_proj, v_proj | 8-16 |
| 新知识注入 | 所有线性层（重点 FFN） | 16-64 |
| 两者都要 | 全部 | 16-32 |

## 训练 vs 推理显存对比

| 阶段 | 7B 模型显存需求 | 说明 |
|------|----------------|------|
| 推理（FP16） | ~14 GB | 仅加载权重 |
| 推理（INT4） | ~4 GB | 量化后 |
| 全量微调（FP16） | ~56 GB | 优化器状态 + 梯度 + 激活值 |
| LoRA 微调（FP16） | ~16 GB | 冻结原权重，只训练 Adapter |
