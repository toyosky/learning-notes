# LoRA 微调入门 - 大模型低秩适配

## 核心思想

大模型微调时，权重变化量其实是**低秩**的——不需要改所有参数，只需在一个低维子空间里调整。

> **类比**：一本 1000 页的百科全书，你只需要在 10 页上贴便利贴，而不是重印整本书。

数学上：$W' = W + \Delta W = W + B \times A$

- $W$：原始权重矩阵（冻结不动）— $d \times d$
- $A$：降维矩阵 — $r \times d$（随机初始化，$r \ll d$）
- $B$：升维矩阵 — $d \times r$（全零初始化）
- 训练时只更新 $A$ 和 $B$，参数量从 $d^2$ 降到 $2dr$

## 来源

> Aghajanyan et al. (2020) 发现：微调后权重变化量的**内在维度**远低于参数空间维度。这是一个**实证发现**，不是严格数学证明。Hu et al. (2021) 据此提出 LoRA。

## 关键超参数

| 参数 | 含义 | 典型值 | 怎么选 |
|------|------|--------|--------|
| **r（秩）** | 低秩维度，控制学习容量 | 8-64 | 简单任务 4-8，复杂任务 16-64 |
| **alpha** | 缩放系数，控制 LoRA 影响力 | r 的 1-2 倍 | alpha/r = 1（标准），2（放大） |
| **target_modules** | 对哪些层加 LoRA | q_proj, v_proj | 见下 |

### alpha 与 r 的互动

实际更新量 = (alpha / r) × B × A

- r=8, alpha=16 → 缩放 2×，学习信号放大
- r=16, alpha=16 → 缩放 1×，标准行为
- alpha 固定，r 增大 → 每个方向分到的更新量变小

## target_modules 选哪些层

### 模型内部结构

```
输入 → [Attention 层] → [FFN 层] → 输出
        ├─ q_proj（查询）
        ├─ k_proj（键）
        ├─ v_proj（值）
        └─ o_proj（输出整合）
        ├─ gate_proj（门控）
        ├─ up_proj（升维）
        └─ down_proj（降维）
```

### 推荐策略

| 策略 | target_modules | 适用场景 |
|------|---------------|---------|
| **最小** | q_proj, v_proj | 风格迁移、简单指令跟随 |
| **推荐** | q_proj, k_proj, v_proj, o_proj | 大多数下游任务 |
| **全面** | 所有线性层（含 FFN） | 新知识注入、领域适应 |

## 实战代码（HuggingFace PEFT）

```python
from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
)

model = get_peft_model(base_model, lora_config)
model.print_trainable_parameters()
# trainable params: 8.3M / 7,000M = 0.12%
```

## LoRA 技术与对比

| 技术 | 原理 | 额外存储 | 效果 |
|------|------|---------|------|
| **LoRA** | 低秩分解 ΔW = B × A | 0.1%-1% | 接近全量微调 |
| **QLoRA** | LoRA + 4-bit 量化基座 | 0.1%-1% | 省显存 4×，效果略低 |
| **DoRA** | 权重分解 = 方向 × 幅度 | 0.1%-1% | 比 LoRA 更稳定 |
| **AdaLoRA** | 自适应分配 r | 0.1%-1% | 自动学习每层应该用多大 r |
| **全量微调** | 更新所有权重 | 100% | 效果最好但最贵 |

> QLoRA 让单卡 24GB 可以微调 7B 模型，是目前最实用的方案。

## LoRA 配置速查（按模型规模）

| 模型大小 | 推荐 r | 推荐 target_modules | 训练显存（FP16） |
|---------|--------|-------------------|----------------|
| 7B | 16 | q_proj, v_proj | ~16 GB |
| 13B | 16-32 | q, v, k, o | ~28 GB |
| 70B | 32-64 | 全部线性层 | ~140 GB（需多卡） |
